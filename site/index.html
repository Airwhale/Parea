Agentic AI Hackathon 0605

Parea Wander

A chat-first group adventure agent that plans a live route, detects when the group leaves the current zone, revises memory, and sends a reroute back into the conversation.

GitHub Repo Judge Runbook Demo Deck Integration PR
Interface Photon/Spectrum terminal or Slack chat
Persistence Butterbase app tables for sessions and adventures
Memory XTrace belief seed, contradiction, and read-back
Generation RocketRide pipeline using live venue candidates
Connected to Butterbase cloud chat.

How Judges Communicate With It

Judges can type directly into the chat window on this page. The terminal and Slack demos use the same message script for local judging. 

npm run judges:demo

I am in
we want something mellow
we moved to Chinatown
where are we?

Slash commands also work: /join, /vibe mellow, /move chinatown, and /status.

Architecture

The engine stays deterministic for safety-sensitive decisions. The LLM writes adventure content, while code checks zone exit and persistence contracts. 

flowchart LR
  User["Judge or group chat"] --> Spectrum["Photon Spectrum
Terminal or Slack"]
  Spectrum --> Commands["Command controller"]
  Commands --> Engine["Parea engine"]
  Engine --> Geo["Geo module
centroid, noise, zone exit"]
  Engine --> Store["Store interface"]
  Store --> Butterbase["Butterbase tables"]
  Engine --> Memory["Memory interface"]
  Memory --> XTrace["XTrace beliefs"]
  Engine --> Generator["Adventure generator"]
  Generator --> RocketRide["RocketRide pipeline"]
  Generator --> Venues["Overpass venues"]
  Engine --> Spectrum

Privacy Rule

Individual GPS points are used transiently. Only the generated adventure zone is persisted.
